## 4. 性能与压力观测 (如有)

结合 E2E 测试过程中对系统响应时间的观测记录（例如接口耗时、页面加载时间等），在此记录系统的异常延迟。

- **高并发响应**：模拟2个并发用户在5秒内持续提交时，因前置的 `toBeVisible` 语言选择器元素查找超时导致会话阻断。未能成功收集到关于 `P95` 响应耗时的指标，需要先修复页面渲染和选择器定位的问题。

## 5. 测试结论与建议

### 5.1 测试结论
本次系统 E2E 测试共执行 28 条核心场景，通过率为 67.8%。其中核心的“正常提交代码获得 Accepted”、“Time Limit Exceeded”、“Wrong Answer”、“Compilation Error”等测试用例均通过，说明后端评测核心机能运作正常。但涉及到登录组件、多语言切换、内存限制判定等用例未通过。

### 5.2 优化建议
1. **修复登录和表单选择器定位**：前端页面可能修改了 ID 导致 `locator('#login-username')` 无法找到，建议开发检查前端 DOM 结构并与测试脚本对齐。
2. **审查内存超限 (MLE) 的沙箱隔离**：测试表明申请超大内存未能返回预期的“内存超限”错误，建议检查 Linux cgroups 或 setrlimit 针对内存的限制是否在沙箱中成功应用。
3. **更新视觉基准快照**：如果页面 UI 有计划内的调整，请使用 Playwright 更新视觉对比快照基准线。
